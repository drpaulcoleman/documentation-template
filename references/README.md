# references/

A **local-only working area** for reference material that informs your
documentation build. Everything here except this `README.md` is git-ignored —
it is never committed and never published to GitHub Pages.

## What goes here

- Web pages and PDFs pulled in by the **`/download`** command (via the
  `reference-harvester` skill) — saved with their source URL and retrieval date
  so they can be cited correctly later.
- `PROJECT-PLAN.md` — the task-tracked build/refinement plan for this repo.
- Any notes, source documents, or research you want an AI assistant to read for
  context while building or revising `index.html`.

## Why local-only

Reference material is usually **someone else's copyrighted content**. Keeping it
git-ignored means you can aggregate it freely for AI context and accurate
citation **without republishing it**. When you quote or paraphrase any of it in
`index.html`, attribute it in the References section using the APA-style entry
template (with a "Retrieved [date]" marker) — see the AI-build notes inside
`index.html`.

## Citation hygiene

Use this material to inform and *cite* your writing — not to copy it. Direct
quotes must be quoted and attributed; everything else should be your own
synthesis. The template's References section exists to make that easy.
