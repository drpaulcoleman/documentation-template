# docs/

A general-purpose folder for anything you want to **keep in the repository**
alongside `index.html` — and have published to GitHub Pages.

Good candidates:

- Code samples or a reference implementation that your documentation discusses
- Downloadable assets (diagrams, spreadsheets, sample data, slide decks)
- Supplementary pages you link to from `index.html`

This folder **is committed and published** (unlike `references/`, which is a
local-only working area that is never published).

### Don't need it?

Delete the whole `docs/` folder. Nothing else in the template depends on it.
