Bottom Line Up Front

What this document is

This is a generative AI documentation-template shell. On its own it renders as a working demo of every feature a finished document gets: sidebar navigation with scroll-spy, full-text search, browser-history-aware deep links, paragraph bookmarking, Mermaid diagrams, an APA references section, dual reading modes, and an AI-coach helper. Run /initialize-project to replace this shell with your own subject matter.

Executive view. This shorter view is for readers who want the conclusion without the supporting detail. Mark exec-only content with data-view-only="exec" and detail-only content with data-view-only="tech".

Open in Claude Code — then use these commands

Claude Code runs in VS Code, Cursor, JetBrains (sidebar extension), or any terminal (claude CLI). Once open in your cloned repo, the following slash commands are available:

/plan <subject> Sets up the project (layout, metadata, git hooks, README), optionally downloads references, then researches the subject and proposes an approved section outline. Saves the plan to references/CONTENT-PLAN.md. Re-run any time to restructure — does not overwrite content.
/download <url or topic> Harvests web pages, PDFs, and search results (including Google Scholar) into references/. Run as many times as you like — Claude reads everything there as context when writing.
/write [section] Generates full prose, diagrams, and citations from the plan and downloaded references. Omit the argument to write everything; pass a section id to rebuild just that section. Updates nav, ai-content-map, and llms.txt.
/check Verifies factual claims, checks for dead or stale citations, and flags coverage gaps. Offers a cross-model fact-check if other AI CLIs are installed.
/enhance-document Audits UX, design, and world-class quality — feature utilisation, theme appropriateness, information architecture, and "what would make this truly outstanding?" Offers a cross-model design review and applies quick wins.

Typical Claude Code session: /plan → approve the outline → /download more sources → /write → review in browser → /check → /enhance-document → commit and push. You direct; Claude authors. See the Prompting guide for conversation examples.

Section 1

Getting Started

Replace this section with your own content. Each major section is a <section class="card" id="..."> with a matching sidebar link. Keep ids short, lowercase, and hyphenated — they become the deep-link URL hash.

Reader features included

  • Deep links & history — every section and bookmarked paragraph is addressable; browser back/forward retrace the reading trail.
  • Search — press / to search; Enter / Shift+Enter cycle matches; Esc clears.
  • Persistent checklists — checkbox state is saved per reader in localStorage.

Authoring Content

Any paragraph given an id and the class anchored gets a hover bookmark handle (the ¶ to the left) for paragraph-level deep linking.

To add a section: copy the AI-TEMPLATE:content-section skeleton (in an HTML comment below this card), give it a unique id, add a sidebar link using the AI-TEMPLATE:toc-entry pattern, and write your content.

Citation pattern

Section 4

Prompting guide — talking to Claude about your document

Every improvement to the document happens through conversation. These are proven prompts for each stage of the workflow — copy, adapt, and combine them. Replace [section] with an actual section id (e.g. cost-model), and [subject] with your document's topic.

Starting a new document

What to typeWhat happens
/plan [your subject]Sets up the project, downloads optional references, and designs an approved section outline. Nothing is written to the document yet.
/download [url or topic]Harvests web pages, PDFs, and Google Scholar results into references/. Run as many times as you like before writing.
/writeGenerates the full document from the approved plan and all downloaded references. Run after /plan.
Write the [section] section — focus on the practical implications for a non-technical reader.Writes or rewrites a single section with a specific audience or angle.

Expanding and deepening content

What to typeWhat happens
The [section] section is too thin. Expand it to at least 600 words, add two concrete examples, and cite your sources.Fleshes out a thin section with evidence and examples.
Add an executive summary paragraph to [section] — three sentences, conclusion-first, no jargon.Adds a dual-view exec variant to a section.
Add a Mermaid diagram to [section] showing the relationship between [concept A] and [concept B].Generates a clickable relationship or concept-map diagram and inserts it.
Add a Mermaid flowchart to [section] showing the decision process for [scenario].Generates a decision-tree or process-flow diagram with yes/no branches.
Add a Mermaid sequence diagram to [section] showing how [actor A], [actor B], and [actor C] interact during [process].Generates a message-flow diagram — great for API calls, auth flows, or multi-system processes.
Add a Mermaid architecture diagram to [section] showing the components of [system] and how data flows between them.Generates a high-level system or data-flow diagram.
The diagram in [section] is hard to read. Simplify it — no more than 6 nodes, short labels, and make key nodes clickable links to the relevant sections.Refactors an overcrowded diagram for clarity and adds deep-link navigation.
Add a checklist to [section] that a practitioner can follow step by step.Creates a persistent, checkable task list.
Add a comparison table to [section] contrasting [option A] and [option B] across: cost, complexity, scalability, and risk.Generates a structured comparison table with your chosen dimensions.
/download [url] — then update [section] with insights from that source.Harvests a new reference and immediately incorporates it, adding the APA citation.

Adjusting tone and audience

What to typeWhat happens
Rewrite [section] for a C-suite audience — lead with business impact, cut technical detail by half.Shifts register without losing substance.
Rewrite [section] for a hands-on practitioner who will implement this themselves. Add specific steps, commands, and gotchas.Makes abstract guidance concrete and actionable for doers.
Rewrite [section] for a sales engineer who needs to explain [subject] to prospects without going deep technically.Shifts to benefit-led language with analogies instead of mechanics.
Rewrite [section] for a skeptical audience — anticipate objections and address the top three directly.Adds counter-argument handling and evidence-based reassurance.
Simplify [section] to a reading level suitable for a smart high-school student with no background in [subject]. Replace every acronym, define every term on first use, and use everyday analogies.Dramatically lowers the reading barrier for a broad or mixed audience.
The document mixes formal and conversational tone inconsistently. Standardise the whole document to a professional but approachable register — clear sentences, active voice, no passive constructions.Runs a consistency pass across all sections for voice and tone.
Make [section] more concise — cut it by 30% without losing the key points.Tightens verbose prose.
The [section] section uses too much jargon. Rewrite it for someone new to [subject].Makes a dense section accessible without dumbing it down.
Add a callout box to [section] highlighting the single most important takeaway.Inserts a styled callout with the key insight.

Restructuring

What to typeWhat happens
I want to add a new section between [section-a] and [section-b] covering [topic]. Write it and update the sidebar.Inserts a new section and keeps nav in sync.
/plan (re-run on existing doc)Re-researches the subject and proposes a revised structure without touching existing content. Use when the scope of the document has grown.
Split [section] into two sections: one covering [topic A] and one covering [topic B].Divides an overloaded section cleanly.
Merge [section-a] and [section-b] into a single section. Keep the best material from both.Consolidates thin or overlapping sections.

Citations and references

What to typeWhat happens
Review the whole document and add citations anywhere a factual claim is made without one.Systematic citation pass across all sections.
